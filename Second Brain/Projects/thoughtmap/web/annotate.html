Jak anotować?

Decyzje

  • Verify — typ jest OK, encja jest prawdziwa
  • Reject — to nie jest encja (fraza, śmieć, boilerplate)
  • Retype — encja jest prawdziwa, ale ma zły typ
  • Alias — ta nazwa to forma innej encji
  • Merge — ta encja powinna zostać złączona z inną
  • Unsure — nie wiesz, zostaw do sprawdzenia

Typy encji

  • person — imię i nazwisko człowieka
  • organization — firma, instytucja, NGO
  • project — projekt, produkt, inicjatywa
  • tool — oprogramowanie, framework, API
  • location — miasto, kraj, adres
  • concept — abstrakcyjny termin
  • other — inne

Przykłady błędów

  • Adevait → location ❌ → powinno być organization
  • B.Braun → location ❌ → powinno być organization
  • "acceptance criteria" → encja? ❌ → Reject (boilerplate)
